Throw away code

August 22, 2009

Let’s start simple. Here is a situation I have encountered more than once. We have this big long class or method, and embedded in it are several blocks of code, all commented out. def some_method(a) do_something(a) calc_another # wow # do_do # calc_calc # zcalc(z) end What should I do? Given the code is version controlled, I would happily throw it away. The code is commented out and will have no effect whatsoever on any program or user.

selected problems in rails?

August 13, 2008

One thing I found out that about rails is that the select_tag is a pain. Why? Because this would not work: <%= select_tag ‘industry’, options_from_collection_for_select(@industry_list, “id”, “name”, @industry) %> But this would work: <%= select_tag ‘industry’, options_from_collection_for_select(@industry_list, “id”, “name”, @industry.to_i) %> Well, that’s for forgetting that HTTP query parameters are strings!
